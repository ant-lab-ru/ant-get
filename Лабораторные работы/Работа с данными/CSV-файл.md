> [!INFO] Определение
> **CSV‑файл** (Comma‑Separated Values — «значения, разделённые запятыми») — это **текстовый файл для хранения табличных данных** в простом, универсальном формате.

# Как устроен CSV

- Каждая **строка файла** соответствует одной строке таблицы.
- Значения в ячейках **разделяются специальным символом‑разделителем** (чаще запятой, но бывают и другие варианты).
- Первая строка **может содержать заголовки столбцов** (но это не обязательно).
- Все данные хранятся как **текст** — числа, даты и прочие типы нужно преобразовывать при чтении.

### Пример CSV‑файла

```
Имя,Возраст,Город
Алексей,28,Москва
Елена,34,Санкт-Петербург
Дмитрий,42,Казань
```

Здесь:

- 3 столбца: «Имя», «Возраст», «Город»;
- 2 строки с данными;
- разделитель — запятая.

# Ключевые особенности

1. **Текстовый формат**  
    Файл можно открыть любым текстовым редактором (Notepad, VS Code и т. п.).
2. **Гибкость разделителей**  
    Помимо запятой, используют:
    - точку с запятой (`;`);
    - табуляцию (`\t`) — тогда файл называют TSV;
    - другие символы (настраивается в программах).
3. **Экранирование значений**  
    Если значение содержит разделитель или перенос строки, его заключают в **двойные кавычки**:
    
```
"Москва, Россия",25
```

4. **Кодировка**  
    Рекомендуемая кодировка — UTF‑8 (поддерживает кириллицу и спецсимволы).
5. **Отсутствие форматирования**  
    В CSV нет:
    - шрифтов, цветов, границ;
    - формул и ссылок;
    - многоуровневых структур (только плоская таблица).

# Где применяется

- **Обмен данными** между системами (базы данных, CRM, ERP).
- **Экспорт/импорт** в таблицах (Excel, Google Sheets).
- **Хранение логов** и простых наборов данных.
- **Подготовка данных** для машинного обучения (часто как промежуточный формат).
- **Резервное копирование** табличных данных.

# Как открыть CSV‑файл

1. **Табличные редакторы**:
    - Microsoft Excel;
    - Google Sheets;
    - LibreOffice Calc.
2. **Текстовые редакторы** (для просмотра структуры).
3. **Языки программирования** (Python с `pandas`, R и др.).

# Плюсы CSV

- **Универсальность**: поддерживается почти всеми программами.
- **Простота**: легко читать и редактировать вручную.
- **Компактность**: занимает меньше места, чем Excel‑файлы.
- **Независимость**: не требует проприетарного ПО.

# Минусы CSV

- **Нет типизации**: все значения — текст (нужно конвертировать).
- **Ограниченная структура**: не поддерживает вложенные данные.
- **Проблемы с кодировкой**: если не указать UTF‑8, могут появиться «кракозябры».
- **Ошибки при экранировании**: некорректные кавычки ломают структуру.

# Стандарт RFC 4180

Существует официальный стандарт **RFC 4180**, который регламентирует:
- использование CRLF (`\r\n`) для переноса строк;
- обязательное количество столбцов в каждой строке;
- правила экранирования кавычек (двойные кавычки внутри значения дублируются: `""`).

> [!NOTE] Итог
> CSV — простой и универсальный формат для табличных данных, идеальный для обмена информацией между разными системами, но требующий внимания к кодировке и экранированию.
